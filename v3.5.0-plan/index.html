    五、TRAE 执行清单

    按优先级排列，TRAE 在 handoff 分支上逐项完成：

    Task V1: Shell 自动补全（优先做）

    文件：cmd/newapi/main.go + 新增 cmd/completion/
    方式：Cobra 内置 completion 命令

    // 在 rootCmd 上注册 completion 子命令
var completionCmd = &cobra.Command{
    Use:   "completion [bash|zsh|fish|powershell]",
    Short: "Generate shell completion script",
    Long:  `Output shell completion code for the specified shell.`,
}

func init() {
    rootCmd.AddCommand(completionCmd)
    completionCmd.AddCommand(&cobra.Command{
        Use:   "bash",
        Short: "Generate bash completion",
        RunE: func(cmd *cobra.Command, args []string) error {
            return rootCmd.GenBashCompletion(os.Stdout)
        },
    })
    // zsh, fish, powershell similarly
}

    测试：go build ./... + 运行 newapi-tools completion bash 确认输出

    Task V2: Docker API 集成评估

    文件：internal/docker/http_client.go + internal/docker/sdk_client.go
    目标：评估 TRAE 编写的 HTTP 客户端是否可以替代现有 exec.Command

    2. 编写集成测试，在 Linux 实机验证 HTTP 端点连通性
    3. 输出评估报告：是否可以替换 / 风险点 / 性能对比

    Task V3: 新增 config validate 子命令

    文件：internal/cli/config.go
    工作量：⭐

    基于 core.NewAPIConfig.Validate() 已有实现，新增 CLI 入口：

    var configValidateCmd = &cobra.Command{
    Use:   "validate",
    Short: "Validate configuration",
    RunE:  runConfigValidate,
}

func runConfigValidate(cmd *cobra.Command, args []string) error {
    cfg := core.GetConfig()
    if cfg == nil {
        return fmt.Errorf("configuration not loaded")
    }
    if err := cfg.NewAPI.Validate(); err != nil {
        fmt.Printf("❌ Configuration invalid: %v\n", err)
        return err
    }
    fmt.Println("✅ Configuration is valid.")
    return nil
}
