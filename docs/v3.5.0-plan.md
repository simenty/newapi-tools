## 二、候选方向

按优先级从高到低排列：

### A. 工程基建加固

| 方向 | 内容 | 工作量 |
|------|------|--------|
| A1 | 补齐 cli 测试至 ≥30% | ⭐⭐⭐ |
| A2 | 补齐 osutil 测试至 ≥30%（当前 28.4%） | ⭐ |
| A3 | CI 中增加 `go mod tidy` 检查，防止 go.sum 漂移 | ⭐ |
| A4 | 新增 `.github/dependabot.yml` 自动检测依赖更新 | ⭐ |

### B. Docker API 客户端集成

TRAE 已实现 `internal/docker/http_client.go`（基于 net/http）和 `sdk_client.go`（接口定义）。  
v3.5.0 可正式集成：

| 步骤 | 内容 | 说明 |
|------|------|------|
| B1 | 确认 http_client 覆盖足够场景 | 与现有 exec.Command 并行可用 |
| B2 | 新增 `docker --mode` flag | 切换 CLI / HTTP 模式 |
| B3 | 逐步迁移 ContainerList / ImagePull | 从 exec 迁移到 HTTP API |
| B4 | 测试双模式一致 | 确保两种路径输出相同 |

**优点**：减少 Shell 注入面、更精细的错误信息、不依赖 docker CLI 版本  
**风险**：需要额外授权配置（Docker socket 权限）

### C. 用户体验（CLI）

| 方向 | 内容 | 工作量 |
|------|------|--------|
| C1 | Shell 自动补全（bash/zsh/fish） | ⭐ |
| C2 | `doctor` 增加网络连通性检查项 | ⭐ |
| C3 | `doctor` 增加 Go 版本检查（远程最新版 vs 当前版） | ⭐ |
| C4 | `install --interactive` 增加代理配置步骤 | ⭐ |

Cobra 内置 shell completion 支持（`cmd/handoff.go`）+ `__complete` 命令。  
实现方式：在 rootCmd 上调用 `completionCommand`，生成补全脚本。

### D. 安全加固

| 方向 | 内容 | 工作量 |
|------|------|--------|
| D1 | GitHub Release 产物 GPG 签名 | ⭐ |
| D2 | `update --self` 启用 RequireSHA256（v3.3.5 已实现） | ⭐ |
| D3 | docs/security.md 安全最佳实践文档 | ⭐ |

## 三、推荐路线

```
v3.4.1 ← 当前
  │
  ├── A1+A2 (测试补齐) → 基础质量
  ├── C1 (自动补全) → 高频体验
  ├── B1+B2 (Docker API 评估) → 技术升级
  └── C2+C3 (doctor 增强) → 诊断能力
  │
v3.5.0 ← 目标
```

### 建议优先级排序

```
P0: A2 (osutil 测试) → 最快，28.4%→30% 门槛
P0: A1 (cli 测试) → 17.4%→30%，影响覆盖率
P0: A3 (go mod tidy CI) → 预防性
P1: C1 (自动补全) → 高频功能，用户感知强
P1: C2 (doctor 网络检查) → 诊断增强
P2: B1+B2 (Docker API 评估) → 不急于替换
P2: D1+D2 (签名+校验) → 安全加分
```
