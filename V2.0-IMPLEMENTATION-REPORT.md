## 一、项目概述

### 1.1 项目背景

NewAPI Tools 是一个用于自动化部署和管理 NewAPI 全家桶（NewAPI + MySQL + Redis + Nginx Proxy Manager）的 Shell 脚本工具集。

### 1.2 V2.0 主要改进

V2.0 版本带来了全面的架构升级和安全增强：

| 类别 | V1.x | V2.0 |
|------|------|------|
| 架构 | 分散式 | 模块化 + 库文件 |
| 配置 | 硬编码 | YAML 配置中心 |
| 状态管理 | 无 | 完整状态跟踪 |
| UI | 基础 | 新手/专家双模式 |
| 安全 | 基础 | 全方位安全加固 |
| 测试 | 无 | 单元测试覆盖 |
| 回滚 | 无 | 完整回滚机制 |

## 二、架构改进

### 2.1 模块化架构

```
newapi-tools/
├── lib/                    # 核心库
│   ├── common.sh          # 通用函数（日志、错误处理）
│   ├── config.sh          # 配置管理
│   ├── state.sh           # 状态管理
│   ├── ui.sh              # UI 组件
│   ├── mode.sh            # 新手/专家模式
│   ├── smart-defaults.sh  # 智能默认值
│   ├── security.sh        # 安全工具库（新增）
│   └── npm-api.sh          # NPM API 封装
├── modules/               # 功能模块
│   ├── deploy/            # 部署模块
│   ├── init/              # 初始化模块
│   ├── manage/            # 管理模块
│   └── monitor/          # 监控模块
├── scripts/              # 工具脚本
├── test/                # 测试文件
├── docs/                # 文档
└── newapi-tools.sh      # 主入口
```

### 2.2 核心库功能

#### lib/common.sh - 通用函数
- 日志记录（log_info, log_warn, log_error, log_debug）
- 依赖检查（check_dependency, require_docker）
- 用户交互（ask_yn, confirm_action）
- 初始化函数（init_toolkit_env）

#### lib/config.sh - 配置管理
- YAML 配置读写
- 配置验证
- 默认值处理
- 安全配置更新（防止注入）

#### lib/state.sh - 状态管理
- 安装步骤跟踪
- 状态持久化
- 步骤完成标记
- 状态恢复

#### lib/ui.sh - UI 组件
- 进度显示（show_progress）
- 摘要展示（show_summary）
- 表格输出（show_table）
- 颜色支持（UI_SUCCESS, UI_ERROR 等）

#### lib/mode.sh - 新手/专家模式
- 智能检测用户类型
- 新手提示
- 专家快捷操作
- 模式切换

#### lib/security.sh - 安全工具库
- 密码管理
- 文件安全
- 输入验证
- 下载校验
- 日志脱敏

## 五、配置系统

### 5.1 配置结构 (config/config.yaml)

```yaml
deploy:
  mysql:
    root_password: ""      # 自动生成
    port: 3306
  redis:
    password: ""          # 自动生成
    port: 6379
  newapi:
    port: 3000
  npm:
    port: 80
    https_port: 443
    admin_port: 81

backup:
  dir: "./backups"
  retention_days: 7

system:
  log_level: "info"
  timezone: "Asia/Shanghai"
```

### 5.2 智能默认值

系统会自动生成安全的默认值：
- MySQL Root 密码
- Redis 密码
- NewAPI 管理员密码
- Session Secret

### 9.3 迁移指南

V1.x 用户迁移到 V2.0：

1. **备份现有数据**
   ```bash
   cd /path/to/newapi-tools
   bash modules/manage/backup.sh --manual
   ```

2. **停止现有服务**
   ```bash
   docker compose down
   ```

3. **更新脚本**
   ```bash
   git pull origin main
   # 或下载最新版本
   ```

4. **重新部署**
   ```bash
   bash newapi-tools.sh
   # 选择重新部署
   ```

5. **验证服务**
   ```bash
   bash modules/monitor/health.sh
   ```
